Added _p24p_write_bool and _p24p_write_ln to src/runtime.spc. write_bool compares argument to 0, prints TRUE or FALSE char-by-char via sys 1. write_ln pushes LF (10) and calls sys 1. Fixed write_int: removed explicit enter/leave (pvmasm .proc/.end handle those automatically), renamed labels to wi_print/wi_done to avoid global namespace conflicts. Created compact test files (under 512-byte pvmasm input_buf limit) that verified correct output on the pv24a VM: TRUE/FALSE for bool, 42/0/-5/7 for int, LF for writeln. Discovered three critical pvmasm constraints: .proc emits enter, main must be first proc (VM starts at addr 0), and input buffer is 512 bytes.
